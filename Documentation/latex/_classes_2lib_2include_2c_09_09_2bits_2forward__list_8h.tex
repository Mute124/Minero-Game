A standard container with linear time access to elements, and fixed time insertion/deletion at any point in the sequence.

Meets the requirements of a \href{tables.html\#65}{\texttt{ container}}, a \href{tables.html\#67}{\texttt{ sequence}}, including the \href{tables.html\#68}{\texttt{ optional sequence requirements}} with the exception of {\ttfamily at} and {\ttfamily operator}\mbox{[}\mbox{]}.

This is a {\itshape singly} {\itshape linked} list. Traversal up the list requires linear time, but adding and removing elements (or {\itshape nodes}) is done in constant time, regardless of where the change takes place. Unlike std\+::vector and std\+::deque, random-\/access iterators are not provided, so subscripting ( {\ttfamily }\mbox{[}\mbox{]} ) access is not allowed. For algorithms which only need sequential access, this lack makes no difference.

Also unlike the other standard containers, std\+::forward\+\_\+list provides specialized algorithms unique to linked lists, such as splicing, sorting, and in-\/place reversal.
