Meets the requirements of a \href{tables.html\#65}{\texttt{ container}}, a \href{tables.html\#66}{\texttt{ reversible container}}, and an \href{tables.html\#69}{\texttt{ associative container}} (using equivalent keys). For a {\ttfamily multimap$<$\+Key,\+T$>$} the key\+\_\+type is Key, the mapped\+\_\+type is T, and the value\+\_\+type is std\+::pair$<$const Key,\+T$>$.

Multimaps support bidirectional iterators.

The private tree data is declared exactly the same way for map and multimap; the distinction is made entirely in how the tree functions are called (\texorpdfstring{$\ast$}{*}\+\_\+unique versus \texorpdfstring{$\ast$}{*}\+\_\+equal, same as the standard).

This turns a red-\/black tree into a \mbox{[}multi\mbox{]}map.
