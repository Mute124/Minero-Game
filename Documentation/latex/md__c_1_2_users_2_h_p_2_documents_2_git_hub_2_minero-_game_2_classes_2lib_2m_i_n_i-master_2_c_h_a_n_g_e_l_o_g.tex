\begin{DoxyItemize}
\item {\ttfamily BUGFIX / FEATURE} Equals ({\ttfamily =}) characters within key names are now allowed. When writing or generating a file, key values containing the {\ttfamily =} characters will be escaped with the {\ttfamily \textbackslash{}=} sequence. Upon reading the file back, the escape sequences will again be converted back to {\ttfamily =}. Values do not use escape sequences and may contain {\ttfamily =} characters.
\item {\ttfamily BUGFIX} Square bracket characters ({\ttfamily \mbox{[}} and {\ttfamily \mbox{]}}) are now valid within section names.
\item {\ttfamily BUGFIX} Trailing comments on section lines are now parsed properly. Fixes a bug where a trailing comment containing the {\ttfamily \mbox{]}} character would break the parser.
\item {\ttfamily BUGFIX} Values being written or generated are now stripped of leading and trailing whitespace to conform to the specified format.
